# 五子棋 · Liquid Glass（Gomoku）

**[English](README.en.md)** · 基于 **React 19、TypeScript、Vite** 实现的浏览器五子棋，采用深色 **Liquid Glass** 界面风格，支持人机对弈、本地历史记录、招式库与棋盘演示。

## 项目简介

这是一个**纯前端**的五子棋应用：无需安装客户端，也没有独立后端，所有对局数据均保存在本地浏览器中。界面采用 Liquid Glass 玻璃拟态风格与渐变背景，棋盘和侧边栏会跟随窗口大小自动适配；在窄屏设备上，主内容区域会改为纵向排列。

## 功能说明

### 人机对弈

- **棋盘规格**：标准 **15×15**，默认**你执黑先行，AI 执白**。
- **简单难度**：会认真对待必败防守等应手，平时下棋则较为随性，偶有意想不到的一步。对候选点按静态估值做 **softmax 温度抽样**；**仅在此难度**提供落子提示光效与棋形评分命名。
- **普通难度**：比简单档算得更深，落子更稳、防守也更「黏」；无提示，适合练手感。采用 **α-β 剪枝**搜索，叶节点使用**邻域启发式**估值。
- **困难难度**：在多个候选点之间反复推演，力求强着；更深的 **α-β** 与根节点 **蒙特卡洛 rollout**，叶节点可选用与文档一致的**全盘线型差分**、候选排序与邻域半径。极端威胁仍会优先堵防，其余更多交给搜索在**攻防之间**权衡。**不包含神经网络**。

### 对局辅助

- **招式板 / 本局走势（简单难度）**：对关键棋形进行打分并命名（例如活三、冲四等），侧边栏会展示当前局势的演变过程。
- **落子提示（简单难度）**：轮到你落子时，可在推荐位置显示高亮标记。
- **胜负判定与连线**：率先连成五子的一方获胜，并高亮显示制胜连线；终局后点击「重新开局」按钮会有强调动效。

### 历史与招式库

- **历史记录**：对局记录保存在浏览器的 **localStorage** 中，可按列表选取查看，支持逐步回放或自动演示，并附带累计胜负等统计数据。
- **招式大全**：内置常见棋形模板，配有棋盘动画与右侧文字解说。
- **从棋形导入续弈**：可将演示中的局面直接带入人机对弈；支持随机模拟至可匹配阵型后再开始行棋；若下一手轮到白方，界面会给出「对方在下 / 轮到你」等明确提示。

### 界面与触控

- 棋盘尺寸会随容器缩放；**窄屏**下限制最大边长、压缩顶部区域高度，并适当减小玻璃面板的内边距，以便腾出更多空间给棋盘网格。
- 落子点与网格线、十字参考线严格对齐；在触控设备上，会在**避免重叠**的前提下适度放大点击热区（详见 `App.css` 中的 `.pt` 规则）。

## 技术栈

- **React 19**、**TypeScript**、**Vite 8**（搭配 `@vitejs/plugin-react`）
- **ESLint**（使用 `typescript-eslint` 及 React Hooks 规则）

### 架构要点

- **三档难度**的 AI 计算均在 **Web Worker**（[`src/ai/ai.worker.ts`](src/ai/ai.worker.ts)）中执行，避免长时间运算阻塞主线程；Worker 不可用时回退到主线程。
- 搜索与估值相关的公用逻辑集中在 [`src/ai/engine.ts`](src/ai/engine.ts) 中。

### AI 行为摘要（延伸阅读）

| 难度 | 核心机制 |
|------|----------|
| **简单** | 强制应对后，对候选点按估值进行 softmax 抽样（温度参数见源码 `EASY_SOFTMAX_TEMPERATURE`）。 |
| **普通** | `pickBestMoveMinimax(board, 3)`；叶节点估值略偏向防守（对方威胁点权重 ×1.08）。 |
| **困难** | `pickBestMoveHardHybrid`：结合 α-β 与根节点蒙特卡洛 rollout；可选用全盘线型差分叶节点估值、快速候选排序、邻域半径 2 等策略。 |

## 本地运行

```bash
npm install
npm run dev
```

终端会输出本地访问地址（通常为 `http://localhost:5173`）。

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动开发服务器（支持热更新） |
| `npm run build` | 执行类型检查并生成生产构建，输出至 `dist/` 目录 |
| `npm run preview` | 本地预览构建结果 |
| `npm run lint` | 运行 ESLint 代码检查 |

## 构建与部署

- 构建输出目录：**`dist/`**
- `vite.config.ts` 中设置了 **`base: './'`**，资源路径均为相对引用，适合部署在 **GitHub Pages** 或其他任意子路径下。

仓库包含 GitHub Actions 工作流（[`.github/workflows/deploy-pages.yml`](.github/workflows/deploy-pages.yml)）：当代码推送至 **`main`** 或 **`master`** 分支时，会自动执行构建并部署。使用 Pages 时，请在仓库 **Settings → Pages** 中将 **Source** 设置为 **GitHub Actions**。

部署后的访问地址一般为：`https://<用户名>.github.io/<仓库名>/`

## 关于本作

### 创作初衷

我希望在浏览器中打造一个具有「Liquid Glass」质感的五子棋界面：深色背景上叠加磨砂玻璃、柔和光效与清晰的功能分区，将人机对弈、历史回放和招式学习集中呈现在同一块画布中。在规则简洁的前提下，把棋形识别、多档难度与棋谱复现做成顺手的一站式体验，同时这也是我练习现代 CSS 与 React 状态管理的一次实践。

### AI 实现与参考

- **简单难度**：AI 会认真防守必输的局面，但平时下棋比较随性，偶尔走出意料之外的步子。本档带有落子提示，推荐位置以高亮标出，适合刚接触五子棋或想轻松下两盘的用户。
- **普通难度**：AI 比简单档算得更深，落子更稳，防守也更黏人；无提示，需自行判断局势，适合想认真练练手感的玩家。
- **困难难度**：AI 会在多个候选位置之间反复推演，力求最优落点，整体棋力最强、较难缠；同样无提示，适合希望挑战自我的玩家。实现请参阅 `src/ai/engine.ts`；计算在 Web Worker 中进行，避免界面卡顿。

若对基于深度强化学习的五子棋 AI 框架（MCTS、PPO、策略-价值网络等）感兴趣，可参考社区开源项目 [guokezhen999/gomoku_rl](https://github.com/guokezhen999/gomoku_rl)（Python / PyTorch，专注于深度学习方向）；它与本作的前端搜索实现相互独立，供算法与工程拓展参考。

经典的 α-β 剪枝五子棋 AI 与教程可参阅 [lihongxun945/gobang](https://github.com/lihongxun945/gobang)（JavaScript；作者说明为传统搜索、不含神经网络，适合对照学习）。
