## 2026-04-10 · 困难算法（参考 Python 文档）

### `src/ai/engine.ts`（困难档搜索链）

- 叶节点可选文档式全局评估：采用 `evaluate_board(白) − evaluate_board(黑)` 的计算方式，按线匹配冲四、活三、眠三等子串模式（与提供的 Python 示例保持一致），并通过 `DOC_GLOBAL_EVAL_SCALE` 压缩数值幅度以适配 α-β 剪枝。
- 走子排序：与文档中的 `quick_score_move` 逻辑一致，对候选点按「落子后该方 `evaluate_board` 值」降序排列，提升剪枝效率。
- 候选范围：困难搜索使用 `radius=2`（与文档 `get_candidate_moves(radius=2)` 一致）；简单难度、普通难度以及入口函数 `chooseAIMove` 仍沿用原有半径。
- 仍保留必堵阈值、蒙特卡洛混合策略，普通档继续使用原有的邻域估值 minimax（未切换为文档叶节点）。

### UI 与困难档 AI 配套调整

- 「关于本作」弹窗：
  - 改用 `createPortal(..., document.body)` 渲染，避免挂载在带有 `transform` 属性的 `.app-root` 下导致 `position: fixed` 宽度异常。
  - 遮罩层改为 flex 居中；卡片最大宽度约 1280px 或 94vw，`z-index` 提高；困难档相关样式与文案同步更新。
- 人机对弈文案：
  - 顶栏难度按钮 `title` 与侧栏 `play-side-desc`：与各档体验口径一致（如必输局面认真防守与随性棋风、无提示、多候选推演等），并与 README、「关于本作」弹窗相互对照。

### 困难档 AI 行为优化（`src/ai/engine.ts`）

- 必堵阈值抬高：仅冲四、活四、成五级别的威胁触发「一步封堵」；活三及以下更多交由 α-β 搜索在攻防之间自行取舍，减少 AI「只会防守」的观感。
- 后续迭代说明：困难档启用文档式全盘线型差分叶节点后，以下加权系数主要影响未走文档叶节点的分支或历史行为记录。混合搜索中内层 minimax 深度设为 4，兜底深度为 5；根候选范围与 band 适当放宽以利于蒙特卡洛模拟。
- 叶节点估值（邻域路径）：采用 `己方最优 × 1.12 − 对方最优 × 0.8` 的加权方式；minimax 增加对方权重与己方加成参数。普通及简单难度仍沿用默认估值权重。

### 文档（同日早期条目）

- 同步更新 `README.md` 与 `README.en.md`：明确三档 AI 的行为差异、延伸阅读资料及外部参考链接；面向访客与 GitHub Pages 试玩用户优化说明结构。
- 简单 / 普通 / 困难基础行为说明：`pickEasySoftmaxSample`（简单）、`pickBestMoveMinimax` 深度 3（普通）、`pickBestMoveHardHybrid`（困难）；简单档提示光由 `pickBestHumanHintMove` 提供。

## 2026-04-11

### 人机对弈 · 白方侧栏与胜负连线

- 侧栏显示优化（`src/App.tsx` + `src/ai/engine.ts`）：
  - AI 落子时展示的分数与招式名称改为调用 `evaluateBoardAtForUi`，该函数仅统计己方连线的棋形（不包含「阻断·…」标记），与黑方的 `evaluateBoardAt` 使用相同启发式尺度，避免因搜索返回极大叶节点值导致白方信息标注异常。
- 引擎层调整：
  - 新增 `evaluateLineSelfOnly` 函数；`evaluateLine` 先统计己方棋形再处理封堵情况。
  - 导出 `evaluateBoardAtForUi` 供 UI 专用。
- 胜负连线渲染（`src/App.tsx` + `src/App.css`）：
  - 终局时 `colDisplayWinLine` 优先使用落子时记录的 `winLine` 信息。
  - 白方获胜时增加 `win-grad-*-onlight` 渐变样式（蓝 → 紫 → 靛），避免浅色渐变叠加在白子上显得发虚；五连白子使用同色系靛紫色描边。
  - 白方胜利时的连线层 `z-index` 适当抬高。
- 胜负连线逻辑加固：
  - 棋谱坐标使用 `clampCoord` + `indexOf` 强制转换为数值，防止 JSON 字符串坐标与 `y*15+x` 拼接出现错位。
  - `resolveDisplayWinLine` 校验存档中的 `winLine` 是否与终局盘面一致，不一致时从盘面反推连线，减少白方获胜后「偶尔不绘制连线」的问题。

### 历史查看功能增强

- 支持按本地日期（`type="date"`）筛选对局记录，提供一键清除日期的快捷操作。
- 列表分页：可选每页显示 8 / 15 / 30 条记录，提供上一页、下一页及页码摘要；当筛选结果为空时显示提示「该日期暂无对局」。

### 移动端棋盘适配

- 棋盘最大边长新增约束条件：同时受视口宽度减去左右内边距的限制，避免横屏时因 `dvh` 与 chrome 计算导致棋盘被过度压缩。
- 窄屏下适当减小 `--app-chrome-y` 变量；低高度横屏时进一步压低顶部 chrome 区域。
- 屏幕宽度 ≤ 960px 时，玻璃面板内边距调整为 15px，为棋盘网格留出更多空间。
- 落子交互优化：添加 `touch-action: manipulation` 并取消 tap 高亮；在粗指针设备上适度放大落子热区（`clamp` 上限为 42px）。

### 招式大全（窄屏适配）

- 招式说明区块与「以此局面续弈」按钮：修复 `catalog-footer` 因 `flex-shrink` / `overflow` 被裁剪的问题。
- `catalog-panel` 支持整体纵向滚动。
- 略微降低卡片区域最小高度、放宽说明区域最大高度限制，并添加底部安全区内边距。

### 人机 AI 估值优化（`src/ai/engine.ts`）

- 估值线段检测窗口由 9 格扩展为 11 格，减少三连棋形贴边时的漏检情况。
- 封堵逻辑增强：在「阻挡对方成五」之后，若黑方（玩家）在某个空点的静态评分超过档位阈值（如眠三及以上），白棋优先占据该位置。
- 略微提高冲四、活三、眠三等对方威胁的权重系数，并相应调整叶节点对方威胁系数（`staticEval` 部分）。

### 文档

  - 分节说明玩法、各档位 AI 行为、触控适配与部署方式。

- `.pt-cross` 改为 `inset: 0`，在落子按钮内部居中绘制十字参考线，确保与网格线精准对齐。
